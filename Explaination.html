1
Webhook trigger
FastAPI fires POST to n8n when a file upload is saved. Payload: document_id, file_path, mime_type, bidder_id.
n8n webhook node
2
Format switch
Switch node routes by mime_type. Native PDF → Docling. Scanned PDF or image → Docling first.
n8n switch node
3
Docling OCR
HTTP POST to Docling microservice. Returns page blocks with text and per-page confidence scores.
Docling :8001
If confidence < 0.70 on any page → route that page to gemma3:4B as base64 image. Merge results back.
4
Vision fallback
Low-confidence pages sent to gemma3:4B via Docker Model Runner OpenAI node. Prompt: extract all text, preserve numbers and dates exactly.
gemma3:4B vision
5
Assemble SDO
Code node merges all page blocks into one Structured Document Object JSON. Stamps source model per block.
code node
6
Embed each block
Split In Batches → each block text sent to nomic-embed-text via OpenAI embeddings node. Returns 768-dim vector.
nomic-embed-text
7
Upsert to Qdrant
HTTP PUT to Qdrant /collections/documents/points. Payload: vector + metadata {document_id, bidder_id, tender_id, page_num, block_type, text}.
Qdrant :6333
8
POST SDO back + audit
HTTP POST to FastAPI /documents/{id}/sdo. Then POST /audit-events with ingestion_complete event.
FastAPI callback
1
Webhook trigger
Officer clicks "Extract Criteria". FastAPI fires POST with tender_id, document_id.
n8n webhook
2
Fetch tender SDO
GET /documents/{id}/sdo from FastAPI. All page blocks returned.
FastAPI GET
3
Flatten + trim
Code node concatenates block texts. If over 12,000 tokens, prioritise sections with "eligibility", "shall", "must", "criteria" keywords.
code node
4
Call mistral-7b
OpenAI node → model-runner.docker.internal. System prompt enforces JSON-only array output. User message: full tender text.
mistral-7b-instruct
Model will wrap JSON in ```json fences. Code node must strip fences before JSON.parse — every time, no exceptions.
5
Parse + validate
Code node strips fences, parses JSON, validates each criterion has required fields. On failure → retry once with stricter prompt.
code + IF node
6
POST registry + audit
POST CriterionRegistry to FastAPI /tenders/{id}/criteria. UI shows criteria list to officer for review. POST audit event.
FastAPI callback
1
Webhook trigger
Officer confirms criteria list. FastAPI fires POST with tender_id, bidder_ids[], criteria[].
n8n webhook
2
Outer loop — bidders
Split In Batches node iterates over each bidder_id one at a time.
split in batches
3
Inner loop — criteria
Nested Split In Batches over each criterion. For each pair: embed criterion text via nomic-embed-text.
nomic-embed-text
4
Qdrant semantic search
HTTP POST to Qdrant search endpoint. Filter: bidder_id = current. Limit: 5 chunks. Returns top-5 most relevant text chunks with metadata.
Qdrant search
Embed criterion.name + criterion.description + criterion.source_text together — not description alone. Richer query = better chunk retrieval.
5
Call mistral-7b for extraction
Grounded extraction prompt: return only what is verbatim in the chunks, null if not found. Must return EvidenceObject JSON.
mistral-7b-instruct
6
Parse EvidenceObject
Strip fences, parse JSON. Validate: value, unit, quoted_text, source_chunk_ref, confidence. null value is valid (means not found).
code node
7
POST evidence + trigger WF-4
POST each EvidenceObject to FastAPI /evidence. After all pairs done → POST to WF-4 webhook to start verdict engine.
FastAPI + WF-4 trigger
1
Fetch all evidence
GET /tenders/{id}/evidence-summary from FastAPI. Returns all EvidenceObjects + CriteriaRegistry for this tender.
FastAPI GET
2
Check for missing evidence
IF evidence.value == null → MANUAL_REVIEW immediately. Reason: "Evidence not found." Skip LLM entirely for missing docs.
IF node
3
Rule layer — no LLM
Pure IF nodes: financial → value ≥ threshold? Certification → expiry_date > submission_date? Compliance → status == "active"? If clear answer AND confidence ≥ 0.80 → verdict set, skip LLM.
IF nodes (pure logic)
Always normalise dates to ISO 8601 in WF-3 before they reach here. Date comparison on raw strings will silently give wrong results.
4
LLM reasoning (ambiguous only)
Technical/semantic criteria always go here. Ambiguous rules or confidence < 0.80 go here. Verdict reasoning prompt: return ELIGIBLE / INELIGIBLE / MANUAL_REVIEW + reason JSON only.
mistral-7b-instruct
5
Confidence gate
Code node: if LLM confidence < 0.75 OR response contains hedging words ("unclear", "possibly", "might", "depends") → override to MANUAL_REVIEW.
code node
6
POST verdict + audit
POST VerdictRecord to FastAPI /verdicts. FastAPI writes append-only to verdict_events. POST AuditEvent per verdict.
FastAPI append-only

Fetch full evaluation

3
Build report structure
Code node organises data: eligible_bidders[], ineligible_bidders[], manual_pending[], per-bidder criterion cards. Verdict cards are built deterministically here — not by LLM.
code node

mistral-7b writes the summary section only. Tables and verdict cards come from deterministic code, not the model. Keeps factual accuracy high.
